# UML Design Lab

## Introduction and Context

In this project, you will work in a team to design a software system based on a real-world scenario.

This activity will take place during the Peer Learning Day (PLD) and is designed to:

- simulate a real collaborative design session
- expose you to multiple valid design approaches
- develop your ability to justify modeling and design decisions

Unlike previous projects, this problem does not have a single correct solution. Your goal is to propose a coherent, consistent, and well-reasoned design.

## Organization

- Teams of 2 to 4 students

- Duration:

- First 2/3 of the day → analysis, design, and preparation

- Final 1/3 of the day → presentations and discussion

Each team will receive a scenario description at the beginning of the session.

## Learning Objectives

By the end of this project, you should be able to:

- analyze a problem and identify system components
- design a consistent object-oriented model
- define responsibilities across classes
- model system behavior using sequence diagrams
- justify design decisions and trade-offs
- evaluate alternative solutions

## General Requirements

- All diagrams must be delivered in Mermaid format
- Use consistent naming across all diagrams
- Your design must be internally consistent
- All modeling decisions must be justifiable
- Avoid unnecessary complexity or over-engineering

## Final Notes

This project is not about finding a single correct solution.

It is about:

- making informed design decisions
- understanding their implications
- learning from alternative approaches

During presentations, you are encouraged to:

- compare your solution with others
- identify strengths and weaknesses
- ask questions and challenge assumptions
